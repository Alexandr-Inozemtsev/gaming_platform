Важное уточнение:
- текущий код Big Walker сейчас основан на 40 клетках (`BigWalkerTokens.cols * rows`)
- reference screens местами показывают декоративную нумерацию, не совпадающую с кодом

- если переводишь игру на 75 клеток, сделай это осознанно и консистентно через tokens + board layout + route painter, не ломая core flow: dice roll -> step-by-step token movement -> next turn -> win on last cell

Технические требования:
- Flutter only
- landscape only
- адаптив для Android + iPhone landscape
- без стороннего game engine
- чистая композиция и поддерживаемый код
- все magic numbers вынести в tokens/constants
- минимально инвазивно по отношению к общей архитектуре

- экран выглядит как production demo, а не прототип
- визуально близок к reference screens
- механика сохранена

- модалки победы/правил/настроек/паузы реализованы
