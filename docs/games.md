# MVP игры: Tile Placement Demo и Roll-and-Write Demo

## Назначение
Документ описывает игровые правила, ботов и взаимодействие клиента с rules-engine для Prompt F.

## Tile Placement Demo
- Поле: сетка 4x4.
- Ход: выбрать пустую клетку и поставить тайл (`A` или `B`).
- Очки: 1 + количество соседних (по стороне) тайлов того же символа.
- UI: drag/drop или tap по клетке, затем confirm (в текущем MVP tap сразу подтверждает).

## Roll-and-Write Demo
- Лист: сетка 5x5 для каждого игрока.
- Бросок: `dice=[d1,d2]` хранится в gameState на сервере.
- Правило: можно отметить только клетку, где `row + col + 2 == d1 + d2`.
- Очки: сумма отмеченных клеток.

## Боты
- `easy`: первый/случайный легальный ход.
- `normal`: простая эвристика
  - tile: выбирает клетку с максимальным мгновенным приростом соседних совпадений;
  - roll-write: выбирает наиболее центральную легальную клетку.

## Персистентность
- Матчи сохраняются в файл (`MATCH_STORE_FILE`) после каждого изменения.
- При перезапуске API состояние матча восстанавливается из файла.

## Acceptance
- Обе игры доступны в соло (против бота) и 1v1 (приватная комната).
- Нелегальные ходы отклоняются сервером через validateMove.
- Восстановление состояния подтверждено тестом persistence.
